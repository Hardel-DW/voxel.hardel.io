# Version 2.0.0 - Voxel Studio

## Bienvenue dans Voxel Studio !
Salut, ça fait longtemps, pratiquement un an voire plus que cette version 2 était en cours. J'ai repoussé mois après mois pour enfin la sortir. Voici tout ce qui arrive et tout ce qui change.

Comme vous le voyez, **Le Configurateur d'enchantement** est renommé **Studio** sur un site séparé. Ce n'est pas cosmétique : ce nouvel outil vise à pouvoir modifier l'ensemble du jeu depuis un seul endroit.

[Découvrir Voxel Studio](https://labs.hardel.io/en-us/studio)

# Nouvelles fonctionnalités
J'ai un tas de nouvelles fonctionnalités à vous présenter. Commençons par les plus visibles : les interfaces et les nouvelles pages.

---

### Interface :
La nouvelle interface est beaucoup plus moderne. L'ancienne était très limitante pour les futures possibilités. Celle-ci n'a pas été pensée juste pour l'instant présent, mais pour les années à venir. Toutes les prochaines fonctionnalités ont été pensées pour être intégrées facilement.

![Recettes](/images/patchnote/studio/studio_edit.webp)

### Nouvelles pages :
- **Studio** supporte maintenant les **recettes** et **loot tables**. Vous pouvez les configurer et les tester directement dans l'interface. J'ai essayé de rendre cela le plus accessible possible.

- Cela arrivera très prochainement : vous pourrez **consulter la liste des modifications** effectuées en haut de la barre latérale, annuler vos changements ou les vérifier.

- L'ajout des pages "Vue d'ensemble" vous permet de consulter facilement l'ensemble des ressources et de les désactiver si besoin.

- L'ajout d'une page de visualisation du code permet de comparer les changements avant et après. C'est très utile pour le débogage.

- Ce n'est pas tout. Pour les enchantements, il est possible de **faire des simulations** concrètes, identiques à celles du jeu, et d'obtenir un tableau de répartition des probabilités pour un objet précis.
-# Cela a nécessité d'éplucher les mathématiques et le code du jeu. C'était dur, avec plus d'une dizaine de paramètres à prendre en compte.

### Les Nouvelles fonctionnalités :
Je vais passer en revue de manière condensée les fonctionnalités. Elles sont nombreuses, je survole ici les plus importantes.

- **Intégration Github** : Il est maintenant possible de récupérer un projet. Vous pouvez faire vos modifications et mettre à jour votre projet en un clic, ou créer une "Pull Request" pour proposer vos ajouts à la communauté.
- **Conversion de Datapack en Mods** : Si vous commencez avec un format .zip, il est maintenant possible de transformer votre datapack en mod, et inversement.
- **Nouvelle barre latérale de recherche** : La nouvelle barre est adaptée à chaque concept. Pour les enchantements, il est possible de rechercher par slots, items ou enchantements non-combinables. C'est extrêmement pratique pour trouver rapidement ce que vous cherchez.
- **La page de recherche d'enchantements** : Cette page accueille de nouveaux paramètres précédemment situés dans la page "Avancé". Certains éléments ont été déplacés vers la page "Avancé".
- **Les Enchantements incompatibles** : Une refonte de son fonctionnement. Vous pouvez maintenant définir si l'enchantement appartient à un groupe ou s'il le cible, ce qui permet d'être au plus proche du fonctionnement du jeu.
-# Concrètement : vous définissez un groupe d'enchantement A, B, C. Par eux-mêmes, ces groupes ne font rien. Ensuite, vous définissez que l'enchantement D cible ce groupe. D sera incompatible avec A, B et C. En théorie, si un enchantement E cible ce même groupe, il sera incompatible avec A, B et C, mais D et E seront compatibles.

# Prochainement :
Le support de tous les concepts manquants : **Advancement**, **Structure**, **Custom Banner**, **Mob Variants**, **Équipement**, **Jukebox Songs** et bien d'autres.

Une fonctionnalité importante : **la mise à jour d'une version à l'autre**, en un seul clic !

**Un éditeur de code intégré**, sous la forme d'un IDE, permettant de modifier directement les fichiers, accompagné d'un éditeur de "Node" associé.

La création de **Patch** : au lieu de modifier le pack, cela créera un nouveau pack appliquant les modifications, vous permettant de mettre à jour l'original sans perdre vos changements.

# Voxel Studio Pro :
On sort du web pour arriver sur votre bureau. C'est le même concept que Voxel Studio, mais avec des fonctionnalités supplémentaires.
- Accès direct à vos dossiers, interagissez directement avec vos fichiers.
- Plus de perte de données si vous rafraîchissez la page. Ici, les données sont stockées sur votre ordinateur.
- Un accès à vos mondes et launchers Modrinth, CurseForge et bien d'autres.
- Des performances bien supérieures. Une énorme quantité d'informations sera stockée sur votre ordinateur, améliorant considérablement la rapidité et la fluidité.

# Technique :
Un an pour ça, ce n'est pas si exceptionnel ?

Tout le futur a été pensé depuis un an : comment intégrer les nouvelles fonctionnalités, visuellement et techniquement. Un bon nombre de technologies ont dû être inventées et je posséderai bientôt 100% de la chaîne. Voici ce que j'ai développé :

- **Breeze** : Le moteur de Voxel Studio. Il a été redéveloppé et est maintenant stable et propre. Il a été réécrit maintes fois pour arriver à une forme que je ne peux plus améliorer.
- **Logger** : Une technologie permettant de connaître les modifications sur n'importe quel concept, de sauvegarder et restaurer les données.
- **Action** : Un nouveau format permettant d'interagir avec les concepts.
- **SNBT** : Une librairie pour naviguer et éditer les fichiers de structure, de joueurs ou du monde. C'est un format propriétaire de Mojang, utile pour la gestion de structures.
- **Differ** : Une technologie pour comparer des fichiers avant/après et remettre les lignes dans l'ordre en préservant les ajouts. Elle permet aussi de connaître les différences entre les fichiers.
- **Markdown** : Cette technologie est sous vos yeux. Elle permet de générer les pages de documentation, la taille du texte, la présentation. Ce que vous lisez actuellement est géré par cette technologie.
- **Parser** : La technologie la plus compliquée du lot. Elle permet de "parser" les datapacks/mods/resourcepacks pour générer un graphique des liens entre les fichiers. Elle permet de savoir quels fichiers dépendent de quoi, et de connaître l'impact d'une suppression.
- **Zip** : Une librairie pour manipuler les fichiers zip et jar (lecture/écriture), réécrite pour le web moderne avec des cadences exceptionnelles.
- **Converter** : Permet de convertir un datapack au format mod.
- **Deploy** : Une technologie pour déployer visuellement ou automatiquement le projet sur Modrinth ou CurseForge. Elle sera intégrée dans Voxel Studio pour les mises à jour après modifications.
- **Renderer** : Un projet permettant d'obtenir tous les visuels des blocs et items de Minecraft dans des fichiers .pog, utilisables pour le web.
- **Atlas** : Permet de générer une image contenant toutes les textures de blocs et items, pour un chargement instantané dans le navigateur.
- **Voxel Api** : Permet la connexion à Github. C'est le serveur de Voxel Studio.

Wow ! Ça fait beaucoup de technologies, mais c'est pour que Voxel Studio soit le plus performant possible. Que manque-t-il concrètement ?
- **Voxel Updater** : Une technologie pour mettre à jour automatiquement les Datapacks/Mods/Resourcepacks d'une version à l'autre. Imaginez passer de la 1.20 à la 1.21.1 en un seul clic.
- **Voxel Structure** : Un algorithme pour placer et générer le visuel d'une structure. C'est un projet complexe, mais qui offrira des tas d'outils.
- **Voxel Rendering** : Ce dernier projet permettra d'afficher des modèles 3D des items et blocs directement dans Voxel Studio. C'est important.
